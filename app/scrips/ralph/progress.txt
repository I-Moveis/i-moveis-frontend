## Codebase Patterns
- Project is a Flutter app; primary working dir is `app/`. Quality gate is `flutter analyze` + `flutter pub get`. Build target for this branch: Android only.
- API base URL is centralized in `lib/core/constants.dart` via `kApiBaseUrl` (compile-time `String.fromEnvironment('API_BASE_URL')`). Override at build time with `--dart-define=API_BASE_URL=...`.
- `absoluteImageUrl(...)` in `lib/core/constants.dart` historically force-upgraded `http://` to `https://`; on `hospedagem_local` esse upgrade foi removido (US-003): URLs http:// e https:// retornam intactas, paths relativos continuam sendo prefixados com a origem de `kApiBaseUrl`.
- Android cleartext traffic is already enabled in `android/app/src/main/AndroidManifest.xml` (`android:usesCleartextTraffic="true"`).
- Branch policy: `development` is sacrosanct on this work — never moved or rewritten. All hospedagem_local work branches off it.
- `lib/core/services/socket_service.dart` deriva a URL do websocket exclusivamente de `kApiBaseUrl` (parse com `Uri.tryParse` + `${scheme}://${authority}` em `_wsUrl`, linhas 39-43). Logo, socket migra automaticamente quando `kApiBaseUrl` muda — nao tem URL hardcoded.
- Runtime validation Android emulator: `adb -s <emu> reverse tcp:3000 tcp:3000` mapeia localhost:3000 do device para o host. Combinado com `kApiBaseUrl=http://localhost:3000/api/` faz o app rodar sem `--dart-define`. Alternativa: usar `--dart-define=API_BASE_URL=http://10.0.2.2:3000/api/` (alias do host pelo emulador), sem precisar do `adb reverse`.
- Flutter Dio interceptor loga requests/responses no formato `→ GET <url>` / `← <status> <url>` e o socket loga `[Socket] ✅ conectado a <origem>`. Use esses prefixos para validar end-to-end por logcat sem instrumentar codigo.
- Package id Android: `com.imoveis.app` (ver `pm list packages` no emulador para confirmar antes de targetar com `monkey`/`am start`).
