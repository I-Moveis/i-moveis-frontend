# Product Guidelines

## Core Principles
1. **Concise & Direct Tone:** All copy, labels, and instructions within the application must be brief, clear, and to the point. We value the user's time and avoid unnecessary fluff or jargon.
2. **Helpful & Actionable Errors:** When an error occurs, the application must clearly explain what went wrong using plain language and provide immediate, actionable steps the user can take to resolve the issue. Avoid exposing raw technical details unless necessary for support.
3. **Speed meets Discovery:** The User Experience must strike a balance between efficiency and delight. Optimize core flows (like searching, filtering, and booking) for rapid task completion, while encouraging exploration through high-quality imagery, rich media, and smooth, purposeful animations.
4. **Standard Accessibility Compliance:** The application must meet standard WCAG guidelines. This includes ensuring adequate color contrast, providing semantic labels for screen readers, and adhering to standard platform accessibility expectations.
